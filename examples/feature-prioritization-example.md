## 4. Trade-Off Documentation

### Top 4 Features — Trade-Off Summaries

**F1: Reasoning Engine**
- *Choosing to:* Invest 8 person-weeks in the AI explanation system.
- *Choosing NOT to:* Ship F5 (PR summaries) or F8 (VS Code extension) this cycle. Those engineers work on reasoning instead.
- *Key assumption:* "Explains its reasoning" converts trial users to paid. If conversion doesn't improve, we over-invested in depth vs. breadth.

**F2: Confidence Scoring**
- *Choosing to:* Ship a threshold UI that lets teams control automation level.
- *Choosing NOT to:* Build this as a simple on/off toggle. The threshold slider is more complex (4 pw vs. 2 pw) but validated as high-value.
- *Key assumption:* Teams actually configure the threshold (vs. leaving the default). Monitor configuration rate in week 1-4.

**F3: GitLab Integration**
- *Choosing to:* Expand platform support to capture a segment Copilot doesn't serve.
- *Choosing NOT to:* Go deeper on GitHub-specific features (GitHub Actions integration, GitHub Projects integration).
- *Key assumption:* GitLab segment has similar pain points and willingness to pay as GitHub segment. If not, we invested 6 pw into a small market.

**F4: Team Learning Model**
- *Choosing to:* Start the learning infrastructure — the moat takes time to build.
- *Choosing NOT to:* Wait for more validation data. We're taking a calculated risk because the moat is strategic.
- *Key assumption:* Teams notice and value the improvement over time. If review quality doesn't visibly improve after 4 weeks, the feature is invisible and the moat doesn't form.

### Deferred Features — Rationale

| Feature | Rationale | Revisit Condition |
|---|---|---|

| F7: Slack notifications | Easy to build (2 pw) but low impact. Quality-of-life, not retention-driving. | Revisit next cycle as a quick win alongside larger features. |

| F10: Auto-merge on green | High risk, low confidence. Automating merges based on AI review creates trust and safety concerns we're not ready to address. Also conflicts with our "AI assists humans" positioning. |
| F11: Bitbucket integration | One request. Insufficient demand signal. |
